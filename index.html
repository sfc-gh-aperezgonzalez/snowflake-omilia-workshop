Load Raw Data

5-12 min

Load 200 synthetic Omilia-style call records from the shared stage. This represents raw production data flowing into Snowflake.

CREATE OR REPLACE TABLE CALLS_RAW (
    CALL_ID VARCHAR,
    CUSTOMER_ID VARCHAR,
    CUSTOMER_NAME VARCHAR,
    CUSTOMER_EMAIL VARCHAR,
    QUEUE VARCHAR,
    LANG VARCHAR,
    CALL_START_TS TIMESTAMP_NTZ,
    CALL_END_TS TIMESTAMP_NTZ,
    DURATION_SEC INTEGER,
    AGENT_ID VARCHAR,

COPY INTO CALLS_RAW
FROM @OMILIA_WORKSHOP_DAY2.SHARED.WORKSHOP_STAGE/calls_raw.csv

Anonymize with AI_REDACT

12-20 min

Remove PII from transcripts before they can be used for training or analytics. AI_REDACT automatically detects and masks names, emails, phone numbers, and addresses.

Production Note In production, this step could be replaced by a call to Omilia's own anonymization model/logic via an external function or UDF. AI_REDACT is used here as a built-in, zero-setup alternative. 
CREATE OR REPLACE TABLE CALLS_ANON AS
SELECT
    CALL_ID, CUSTOMER_ID, QUEUE, LANG,
    CALL_START_TS, CALL_END_TS, DURATION_SEC,
    AGENT_ID, CHANNEL,
    AI_REDACT(RAW_TRANSCRIPT) AS TRANSCRIPT_ANON
FROM CALLS_RAW;
-- Compare: raw vs anonymized
SELECT
    r.CALL_ID,
    LEFT(r.RAW_TRANSCRIPT, 100) AS original,
    LEFT(a.TRANSCRIPT_ANON, 100) AS anonymized
FROM CALLS_RAW r
JOIN CALLS_ANON a ON r.CALL_ID = a.CALL_ID
LIMIT 5;
PII replaced with placeholders like [NAME], [EMAIL], [PHONE_NUMBER].

Enrich with Cortex AI Functions

20-33 min

Use multiple Snowflake Cortex AI Functions to extract structured insights — showcasing AI_COMPLETE, AI_SENTIMENT, AI_CLASSIFY, AI_FILTER, and AI_TRANSLATE.

Timing This step processes 200 rows × multiple AI calls. It takes approximately 3-4 minutes to complete. 
CREATE OR REPLACE TABLE CALLS_ENRICHED AS
SELECT
    *,
    -- AI_COMPLETE: free-form LLM call for summarization
    TRIM(AI_COMPLETE('llama3.3-70b',
        'Summarize this call transcript in exactly one sentence. Only output the summary, nothing else.\n\nTranscript: ' || TRANSCRIPT_ANON
    ), '" \n') AS SUMMARY,

    -- AI_SENTIMENT: purpose-built sentiment extraction
    AI_SENTIMENT(TRANSCRIPT_ANON):categories[0]:sentiment::VARCHAR AS SENTIMENT,

    -- AI_CLASSIFY: multi-class classification with predefined categories
    AI_CLASSIFY(TRANSCRIPT_ANON::VARCHAR,
        ['billing', 'technical_issue', 'cancellation', 'product_inquiry', 'complaint', 'general_inquiry']
    ):labels[0]::VARCHAR AS ISSUE_CATEGORY,

    -- AI_COMPLETE: topic extraction (flexible LLM task)
    TRIM(AI_COMPLETE('llama3.3-70b',
        'Extract the main topics from this call as a comma-separated list (max 3 topics). Only output the topics.\n\nTranscript: ' || TRANSCRIPT_ANON
    ), '" \n') AS TOPICS,

    -- AI_FILTER: boolean classification via natural language predicate
    AI_FILTER(PROMPT('The customer in this call is requesting escalation to a supervisor or manager: {0}', TRANSCRIPT_ANON)) AS ESCALATION_FLAG,

    -- AI_TRANSLATE: translate summary to Greek for Omilia's Athens team
    AI_TRANSLATE(
        TRIM(AI_COMPLETE('llama3.3-70b',
            'Summarize this call in one sentence. Only output the summary.\n\nTranscript: ' || TRANSCRIPT_ANON
        ), '" \n'),
        '', 'el'
    ) AS SUMMARY_EL
FROM CALLS_ANON;
SELECT CALL_ID, SUMMARY, SENTIMENT, ISSUE_CATEGORY, TOPICS, ESCALATION_FLAG, SUMMARY_EL
FROM CALLS_ENRICHED LIMIT 10;
Each row now has a summary, sentiment label, issue category, topics, escalation flag, and a Greek translation of the summary.

External Model Write-Back

33-40 min

Call Omilia's external model (simulated) to score each call. This demonstrates the write-back pattern where model outputs flow back into Snowflake.

What's happening The OMILIA_MODEL_SCORE function simulates an external ML model hosted on AWS. In production, this would be an external function calling a real endpoint. The results (resolution probability, quality score, escalation recommendation) are written back into Snowflake. 
CREATE OR REPLACE TABLE CALLS_TRAINING_READY AS
SELECT
    e.*,
    s.VALUE:resolution_probability::FLOAT AS RESOLUTION_PROBABILITY,
    s.VALUE:handling_quality_score::FLOAT AS HANDLING_QUALITY_SCORE,
    s.VALUE:escalation_recommended::BOOLEAN AS MODEL_ESCALATION_RECOMMENDED,
    s.VALUE:model_version::VARCHAR AS MODEL_VERSION
FROM CALLS_ENRICHED e,
     LATERAL (
         SELECT OMILIA_WORKSHOP_DAY2.SHARED.OMILIA_MODEL_SCORE(
             e.CALL_ID, e.TRANSCRIPT_ANON
         ) AS VALUE
     ) s;
SELECT CALL_ID, QUEUE, SENTIMENT, ISSUE_CATEGORY,
       RESOLUTION_PROBABILITY, HANDLING_QUALITY_SCORE,
       MODEL_ESCALATION_RECOMMENDED, MODEL_VERSION
FROM CALLS_TRAINING_READY LIMIT 10;
CALLS_TRAINING_READY now has all enrichments + external model scores. This is your final training-ready dataset.

Audio Transcription

Bonus

Transcribe pre-staged MP3 files directly in Snowflake with AI_TRANSCRIBE. This demonstrates audio-first data flowing into the same governed pipeline.

Optional Skip this step if you're running short on time. It takes ~1-2 minutes to transcribe 3 audio files. 
CREATE OR REPLACE TABLE AUDIO_TRANSCRIPTIONS_RAW AS
SELECT
    m.CALL_ID,
    m.AUDIO_FILE_NAME,
    AI_TRANSCRIBE(
        TO_FILE('@OMILIA_WORKSHOP_DAY2.SHARED.WORKSHOP_STAGE', m.AUDIO_FILE_NAME),
        {'timestamp_granularity': 'speaker'}
    ) AS TRANSCRIPTION_RESULT,
    CURRENT_TIMESTAMP() AS TRANSCRIBED_AT
FROM (
    SELECT 'CALL-001' AS CALL_ID, 'audiofile1.mp3' AS AUDIO_FILE_NAME
    UNION ALL SELECT 'CALL-002', 'audiofile2.mp3'
    UNION ALL SELECT 'CALL-003', 'audiofile3.mp3'
) m;
SELECT CALL_ID, AUDIO_FILE_NAME,
       TRANSCRIPTION_RESULT:audio_duration::FLOAT AS duration_sec,
       LEFT(TRANSCRIPTION_RESULT:text::VARCHAR, 200) AS transcript_preview
FROM AUDIO_TRANSCRIPTIONS_RAW;
Full transcriptions with speaker diarization stored in AUDIO_TRANSCRIPTIONS_RAW.

Semantic View & Cortex Agent

Bonus

Create a Semantic View for natural-language querying, then a Cortex Agent that answers questions about your data using text-to-SQL.

6A: Create Semantic View

CREATE OR REPLACE SEMANTIC VIEW CALLS_SEMANTIC_VIEW

  TABLES (
    calls AS CALLS_TRAINING_READY
      PRIMARY KEY (CALL_ID)
      COMMENT = 'Omilia call center training-ready data'
  )

  DIMENSIONS (
    calls.queue AS QUEUE
      COMMENT = 'Call queue: billing, technical_support, cancellations, sales, general',
    calls.lang AS LANG
      COMMENT = 'Language of the call: en, el, es',
    calls.channel AS CHANNEL
      COMMENT = 'Channel: phone, virtual_assistant, chat',
    calls.agent_id AS AGENT_ID
      COMMENT = 'Agent who handled the call',
    calls.call_start_ts AS CALL_START_TS
      COMMENT = 'Call start timestamp',
    calls.sentiment AS SENTIMENT
      COMMENT = 'Call sentiment: positive, negative, neutral',
    calls.issue_category AS ISSUE_CATEGORY
      COMMENT = 'Primary issue category',
    calls.escalation_flag AS ESCALATION_FLAG
      COMMENT = 'Whether the call should be escalated (TRUE/FALSE)'
  )

  METRICS (
    calls.total_calls AS COUNT(CALL_ID)
      COMMENT = 'Total number of calls',
    calls.avg_duration AS AVG(DURATION_SEC)
      COMMENT = 'Average call duration in seconds',
    calls.avg_resolution_probability AS AVG(RESOLUTION_PROBABILITY)
      COMMENT = 'Average model-predicted resolution probability (0-1)',
    calls.avg_quality_score AS AVG(HANDLING_QUALITY_SCORE)
      COMMENT = 'Average model-predicted handling quality (0-1)'
  )

  COMMENT = 'Semantic view over Omilia call training data for natural-language analytics';

6B: Create Cortex Agent

CREATE OR REPLACE AGENT CALLS_ANALYST_AGENT
  COMMENT = 'Omilia Call Center Analyst — ask questions about call volumes, sentiment, quality scores, and escalation patterns'
  FROM SPECIFICATION
  $$
  instructions:
    response: |
      You are the Omilia Call Center Analyst. Answer questions about call center operations clearly and concisely.
      When showing data with categories or comparisons, always include a chart visualization.
      Use bar charts for category comparisons and line charts for time trends.
      Format numbers with appropriate precision (percentages to 1 decimal, scores to 2 decimals).

  tools:
    - tool_spec:
        type: "cortex_analyst_text_to_sql"
        name: "CallsAnalyst"
        description: "Queries Omilia call center data including call volumes by queue, sentiment distribution, average handling quality scores, escalation rates, and issue category breakdowns."

  tool_resources:
    CallsAnalyst:
      semantic_view: "CALLS_SEMANTIC_VIEW"
      execution_environment:
        type: "warehouse"
        warehouse: "WORKSHOP_WH"
  $$;

Test Your Agent

Open Snowflake CoWork (top-right corner of Snowsight) and select your CALLS_ANALYST_AGENT. Try asking:

What are the top 3 issue categories by call volume, and what is the average quality score for each?
The agent generates and executes SQL against your semantic view, returning a natural-language answer with charts.
